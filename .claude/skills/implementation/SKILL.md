# 実装スキル

このスキルは、コード実装時のコーディング規約とベストプラクティスを提供します。

## コーディング規約の参照

コードを実装する際は、以下のコーディング規約を参照してください:

```bash
Read('.claude/skills/implementation/coding-standards.md')
```

## 実装の基本方針

### 1. テスト駆動開発 (TDD)

**手順**:
1. テストを書く
2. テストを実行 → 失敗を確認
3. 実装を書く
4. テストを実行 → 成功を確認
5. リファクタリング

### 2. 段階的な実装

**原則**:
- 一度に全てを実装しない
- 小さな単位で動作確認
- コミットは論理的な単位で

### 3. コードレビューの準備

実装完了前のチェック:
- [ ] 全てのテストがパス
- [ ] Lintエラーがない
- [ ] 型チェックがパス
- [ ] コメントが適切に記載されている

## 実装時の注意点

### セキュリティ
- 入力検証を必ず実装
- 機密情報をハードコードしない
- エラーメッセージに機密情報を含めない

### パフォーマンス
- 不要な計算を避ける
- ループ内での重い処理を避ける
- 適切なデータ構造を選択

### 保守性
- 関数は単一の責務を持つ
- マジックナンバーを避ける
- 意味のある命名を心がける
